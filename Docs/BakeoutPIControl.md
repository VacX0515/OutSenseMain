# 베이크아웃 PI 제어 로직 문서

## 개요

베이크아웃 오토런의 온도 제어는 **소프트웨어 PI 피드백**으로 CH1(히터)의 SV를 조절하여 모니터 채널(샘플 온도)을 목표 온도에 도달/유지합니다.

두 가지 모드가 있습니다:
- **CH1 모니터링 (TM4 PID)**: CH1만 선택 시 → TM4 하드웨어 램프+PID로 직접 제어, 소프트웨어 PI 비활성
- **CH2~8 모니터링 (소프트웨어 PI)**: 외부 샘플 센서 선택 시 → 소프트웨어 PI가 CH1 SV를 조절

---

## 실행 흐름

```
RunExperimentAsync()
├── 1단계: 목표 온도 도달 (승온)
│   ├── 소프트웨어 램프: rampedTarget = 초기온도 + 경과시간 × 램프속도
│   ├── PI 피드백 루프 (5초 간격)
│   ├── 안정화 판정 (온도 범위 + 변화율 + SV 안정성 × 600초)
│   └── 도달 완료 → 2단계 전환
└── 2단계: 홀드 (온도 유지)
    ├── 단순 PI (감쇠/에너지보상 없음)
    ├── SV 감소 속도 제한 (3단계)
    └── 설정 시간 경과 → 완료
```

---

## 1단계: 승온 PI 피드백

### 램프 기준선 (rampedTarget)

```
rampedTarget = min(초기샘플온도 + 램프속도(°C/s) × 경과시간, 목표온도)
```

PI의 목표를 `rampedTarget`으로 치환하여 샘플 상승률을 램프 속도 이내로 제한합니다.

### 적응형 게인 스케일링

매 사이클 `observedThermalLag` 기반으로 Kp, Kd를 자동 조정:

```
lagScale = observedThermalLag / 20.0  (클램프: 0.3 ~ 2.0)
Kp = Kp_base(1.5) × √lagScale
Kd = Kd_base(3.0) / √lagScale
```

| 열지연 | lagScale | Kp | Kd | 의미 |
|--------|----------|------|------|------|
| 2°C (빠른 시스템) | 0.3 | 0.82 | 5.48 | 오버슈트 억제 |
| 20°C (기준) | 1.0 | 1.50 | 3.00 | 기본 |
| 40°C (느린 시스템) | 2.0 | 2.12 | 2.12 | 빠른 추종 |

### 열지연 추적 (observedThermalLag)

```
currentLag = max(0, CH1_PV - 샘플온도)

상승 중 (rateOfChange > 0.05): 즉시 반영 (피크 포착)
그 외: EMA 감쇠
  - 목표 근처 (proximity > 0.5): α = 0.005 (느린 감쇠)
  - 그 외: α = 0.02
```

시작 시 초기값은 `CH1_PV - 샘플온도`로 실측합니다.

### PI 출력 계산

```
newCh1SV = rampedTarget + error × Kp + integralTerm + dTerm + energyCompensation
```

| 항 | 계산 | 비고 |
|----|------|------|
| P | `(rampedTarget - 샘플) × Kp` | 비례 |
| I | 누적 `error × Ki_norm(0.01)` | 양: Ki, 음: Ki × 1.5 |
| D | `-Kd × smoothedRate` | 양방향 (상승 억제 + 하강 복구) |
| E | `-storedEnergy × 0.15 × proximity` | storedEnergy > 0일 때만 |

### 적분 관리

| 항목 | 값 |
|------|-----|
| 적분 최소값 | `observedThermalLag × 0.85` (proximity > 0.3) |
| 적분 최대값 | `effectiveMaxTemp - targetTemp` |
| 감쇠 조건 | `storedEnergy > 2.0°C` |
| 감쇠율 상한 | 5%/사이클 |
| 모델 수렴 | 상향만 (추정값 = lag × 0.9) |

### SV 제한

```
상한: min(effectiveMaxTemp, rampedTarget + max(observedThermalLag, 5) × 1.5)
하한: max(rampedTarget + observedThermalLag × 0.85, 샘플온도 - currentLag × 0.3)
```

**SV 상한이 rampedTarget 기반**이므로, 어떤 샘플이든 CH1 SV가 rampedTarget 대비 열지연 × 1.5 이내로 제한됩니다.

### SV 감소 속도 제한 (3단계)

| 샘플 온도 | SV 감소 허용 |
|-----------|-------------|
| < 목표 | **완전 차단** |
| 목표 ~ 목표+허용오차 | 최대 **0.1°C/사이클** |
| > 목표+허용오차 | 최대 **0.3°C/사이클** |

## 안정화 판정

목표 온도 도달 후 홀드 시작 조건:

```
모든 조건이 동시에 충족되어야 하며, 설정 시간(기본 600초) 동안 연속 유지 필요:
1. 샘플 온도 ∈ [목표 - 허용오차, 목표 + 허용오차]
2. 변화율 < 0.3°C/사이클
3. CH1 SV 변동 < 0.5°C/사이클
```

**도중 하나라도 벗어나면 카운터 0으로 리셋**, 처음부터 다시 카운트합니다.

리셋 사유 로그:
- `범위 이탈 (199.7°C)` — 온도가 범위 밖
- `변화율 과대 (0.35°C/cyc)` — 아직 변하는 중
- `SV 변동 과대 (0.8°C)` — PI가 아직 수렴 안 됨

---

## 2단계: 홀드 PI

홀드는 이미 평형 상태이므로 **단순 PI만 사용**합니다:

```
newCh1SV = targetTemp + holdError × Kp + integralTerm + dTerm
```

- 감쇠/에너지보상 **없음**
- 적분: 대칭 속도 (`Ki_norm`)
- D항: 양방향
- SV 감소 3단계 제한 적용 (승온과 동일)

## 열 특성 실시간 표시

오토런 실행 중 `[열특성]` 라인에 표시:

```
[열특성] 열지연:25.3°C  현재Lag:24.1°C  변화율:0.12°C/cyc  시정수:1054초  열저항:3.42  Kp:1.33 Kd:3.37  I:21.5  정상SV:221.5°C
```

| 항목 | 계산 |
|------|------|
| 열지연 | `observedThermalLag` (EMA 추적) |
| 시정수 | `열지연 / 변화율 × 피드백주기` |
| 열저항 | `열지연 / (히터출력% / 10)` |
| 정상SV | `target + observedThermalLag × 0.85` |

## 정체 감지 (Stall Detection)

CH1이 상한에 붙어있는데 샘플이 안 올라가면 열전달 부족으로 판단:

| 조건 | 값 |
|------|-----|
| 진입 | CH1 SV ≥ 상한-1 AND error > 10°C |
| 판정 | 300초(60사이클) 동안 `error × 10%` 미만 상승 |
| 결과 | 안전 종료 실행 |

목표 근처 (error ≤ 10°C)에서는 정체 판정 안 함 — 열평형에 의한 자연 감속.

## 종료 시퀀스

```
1. 히터 CH1 정지
2. 냉각 대기 (펌프 유지, 진공 보호) → CH1 ≤ 벤팅 시작 온도(125°C)
3. 이온게이지 HV OFF
4. 터보펌프 정지 + 감속 대기
5. 게이트 밸브 닫기
6. 드라이펌프 정지
7. 벤트 밸브 열기 + ATM 압력 대기
8. 배기 밸브 열기 + 쿨링 대기 → CH1 ≤ 쿨링 목표(50°C)
9. 배기 밸브 닫기
10. 벤트 밸브 닫기
11. 칠러 PID 정지
12. 최종 상태 확인
```

인터락: 125°C 이상에서 벤트/배기 밸브 열기 차단 (종료 시퀀스에서도 항상 적용)

## 안전 워치독

| 감지 조건 | 자동 동작 |
|-----------|----------|
| 터보 가동 중 + 칠러 정지 | 터보 자동 정지 |
| 터보 가동 중 + 드라이펌프 정지 | 터보 자동 정지 |
| 터보 가동 중 + 게이트밸브 닫힘 | 터보 자동 정지 |
| CH1 > 상한+20°C | 히터 자동 정지 |
| 터보 가동 중 + 압력 > 1 Torr | 터보 자동 정지 |
| 히터 작동 중 + 벤트/배기 열기 시도 | 밸브 차단 |
